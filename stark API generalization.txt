Stark API Generalization
========================
Stark is hardcoded to my problem. However, I should make it general for people to add their own systems.
I don't have to like it, it will be good for adoption.
If someone wants to add micropolar, they are going to need to learn my design and hack in their stuff.
Which is problematic for them and for me as I will not merge it.

I first should focus in having a fully featured simulator, even if hardcoded.
Then I can generalize it.


Plugin system
-------------
The idea is to build a system on top of Stark core that works with plugins.

At the top level we have the "systems": Deformables, RigidBodies...
I need a better name because "Interactions" is not a system though.
This is easier to generalize as we can just have a dict where people can add new sources.
All of them get initialized and no problem.

The problem is the next level of abstraction, which involves interactions.
With no interactions, we are basically done.
Stark already have a system of lambdas which is amazing and let's you do almost anything at any time.
But when interactions are necessary, the user needs to care to make their meshes "seen" by the contact energy system.
The only way to do this really is to make the user declare their meshes *as a new deformable subtype*.
If it is not deformable or rigid body, they will have to figure out what they do for contacts.

This means that we have to generalize Deformables.
We need an interface so that new subsystems keep their dynamics and collision meshes concatenated with the rest.


Further:

How would we incorporate triangle meshes where their vertices are interpolations of the simulation mesh? (FCM, high order)


Final question: Do we really want to make this available for everyone so easily?
Is the *scientific* adoption that useful?
There is an argument to be made for releasing a good open source Stark Core and
a sufficiently good enough actual solver that can reproduce what we see in the paper.
But that implementation can be purposedly not so flexible to not give away that
advantage and keep it for us.
