Documentation
-------------
	* The big contention point of the architecture is that we have a global `PointDynamics`
		When this is explained and justified, everything else becomes much easier to understand.
		This fundamental decision puts some stress on the mid complexity layer (user layer and core layer remain simple).
		In order for Stark developers to leverage the power of automation, complexities related to multiple discretization are moved there,
		where the harm is minimized. The mid-layer is a funnel. The user doesn't know about any of this and the core doesn't know about the user.
	* DoFs must be non-empty at initialization time. Otherwise they will not be added to SymX.
	* It needs to be revisited as building something completely from zero ala hello world.

	* We a tutorial for a simple sim. Otherwise it can be intimidating if jumped directly into the stark code..
	* Explain IPC edge-edge
	* Rigid body constraints
	* CANNOT BE USED WITH TYPE FLOAT. make_vector uses sizeof()
	* Very careful with rigid body declaration an placement:
		- You are extremely engouraged to use primitive inetial tensor that approximate the shape of your objects.
